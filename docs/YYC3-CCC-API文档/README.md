## 🔧 API 技术栈

### API 框架

- **API 框架**：Next.js API Routes
- **API 版本**：v1.0
- **API 协议**：HTTPS
- **API 格式**：RESTful

### 数据格式

- **请求格式**：JSON
- **响应格式**：JSON
- **字符编码**：UTF-8
- **时间格式**：ISO 8601

### 认证方式

- **认证方式**：JWT
- **Token 有效期**：24 小时
- **刷新机制**：支持 Token 刷新
- **权限控制**：基于角色的访问控制（RBAC）

## 📝 API 规范

### 接口设计原则

- **RESTful**：遵循 RESTful 设计原则
- **幂等性**：GET、PUT、DELETE 方法幂等
- **安全性**：所有接口需要认证
- **版本控制**：支持 API 版本管理
- **错误处理**：统一的错误码和错误信息

### 接口命名规范

- **URL 命名**：小写字母和连字符
- **HTTP 方法**：GET、POST、PUT、DELETE
- **资源命名**：复数形式
- **操作命名**：动词形式
