### 目录结构

```
tests/
├── unit/              # 单元测试
│   ├── components/     # 组件测试
│   ├── lib/          # 工具库测试
│   └── services/      # 服务测试
├── integration/       # 集成测试
│   ├── api/          # API 集成测试
│   └── database/     # 数据库集成测试
├── e2e/             # 端到端测试
│   ├── auth.test.ts   # 认证测试
│   └── flow.test.ts   # 流程测试
├── fixtures/         # 测试数据
│   ├── users.json     # 用户数据
│   └── messages.json  # 消息数据
└── setup.ts          # 测试环境配置
```

## 🔧 测试框架

### 技术栈

- **测试框架**：Vitest 2.1.5
- **断言库**：Vitest 内置
- **Mock 库**：Vitest 内置 vi
- **覆盖率工具**：Vitest 内置
- **E2E 框架**：Playwright
- **性能测试**：Lighthouse
- **安全测试**：OWASP ZAP

### 测试命令

```bash
# 运行所有测试
npm run test

# 运行单元测试
npm run test:unit

# 运行集成测试
npm run test:integration

# 运行 E2E 测试
npm run test:e2e

# 生成覆盖率报告
npm run test:coverage

# 监听模式
npm run test:watch
```

## 📝 测试最佳实践

### 测试原则

- **独立性**：每个测试应该独立运行
- **可重复性**：测试结果应该可重复
- **快速性**：测试应该快速执行
- **清晰性**：测试名称应该清晰明确
- **可维护性**：测试代码应该易于维护

### 测试覆盖率

- **单元测试覆盖率**：> 80%
- **集成测试覆盖率**：> 60%
- **E2E 测试覆盖率**：核心流程 100%
- **整体覆盖率**：> 70%
